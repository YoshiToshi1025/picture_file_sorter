# picture_file_sorter デジタルカメラ画像自動整理ツール

## 概要
このプロジェクトでは、デジタルカメラで撮影した画像データを効率よく整理するためのツールを提供します。

## プロジェクトの目的

デジタルカメラで撮影した画像データをどのように整理していますか？
データをとりあえずコピーしたけどどこに置いたかわからなくなったり、記録メディアに残したまま整理できずに放置したりしていませんか？
本プロジェクトで提供するツールを使うと、指定フォルダ下の画像データを、保存用ストレージに、決められたルールのフォルダ構成に仕訳しながら、一気に移動することができます。

## 本ツールの特徴

本ツールは、以下の特徴があります。
- 指定フォルダおよびその下位フォルダに格納されている撮影データを、指定フォルダ以下に移動します。
- 移動先のフォルダ構成は、移動先フォルダ / メーカー名_カメラ名 / 撮影年(YYYY) / 撮影月(01-12) / 撮影日(01-31) になります。 (例: {移動先}/Nikon-Z9/2026/03/17)
- 移動するファイルの種類は、デジタルカメラ（現状、NikonとCanon）のRAW(.NEF, .CR2, .CR3)とJPEG(.JPG, .JPEG)のみです。なお、Nikonデータの場合はRAW（.NEF）データと同名の編集データ(.nksc)があればセットで移動します。
- 移動先に同名のファイルが存在していて撮影日(EXIF)やデータ内容が異なる場合は、ファイル名末尾に枝番("_A"～"_Z")を付けて別ファイルとして保存します。
- 移動する前に移動先のディスクの空き容量が十分に空いていることを確認して、移動処理を開始します。
- 移動先にすでに存在する重複ファイルや、移動対象外のファイルは、移動元のフォルダに残します。
- テスト実行が可能で、どのファイルが移動されるかを事前に確認できます。

* ツールの詳細仕様は、ソフトウェア仕様書(spec.pdf)を参照してください。

## インストール

### 前提条件
- Pythonをローカル環境で実行可能なこと。

### インストール手順
1. Python実行可能環境に、当該プロジェクトのコードをGithub環境からチェックアウトする。
2. チェックアウトフォルダにて、"pip install -r requirements.txt"を実行して、ツール実行に必要なパッケージをインストールする。

## ツールの使い方
1. 移動先フォルダ（保存用ディスクなど）の空き容量を確認します。
2. 移動したいファイルが含まれる記録メディア(SDカードなど)をPCにセットし、アクセス（読み書き）可能にします。
3. コマンド"python sorter.py --src {移動元フォルダ} --dst {移動先フォルダ} --dry-run"で、テスト実行を行います。
　　なお、テスト実行では、実行シミュレーションが行われ実行ログが出力されますが、実ファイルは移動されません。
4. 出力ログを参照して、テスト実行で問題がなかったかを確認します。
5. コマンド"python sorter.py --src {移動元フォルダ} --dst {移動先フォルダ} (--dry-run)"で、ツールを実行します。
6. 処理終了後に、移動元から移動されなかったファイルを確認し、必要に応じて、手動で移動するなどの対応を行います。

## 注意事項
・現状、NikonおよびCanonのカメラで撮影したデータで、EXIF情報付きのものが移動対象となります。その他の写真ファイル（TIFFやHEIF等）や、動画ファイル、EXIF情報（カメラ名や撮影日）の含まれていないファイルは移動されません。
・複数撮影日の写真データをまとめたフォルダに対しては、本ツールではなく手作業での整理をおすすめします。（撮影日で分けられたフォルダにばらばらに移動されてしまいます）

## 対応予定
- 未対応のデータ形式への対応
- 他社カメラデータへの対応
